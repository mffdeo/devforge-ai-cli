Cinco etapas. Um fluxo governado.

1
Init

Inicializa o projeto e a configuração local.

2
Scan

Mapeia stack, risco e áreas sensíveis.

3
Plan

Gera Plan Pack governado, Context Pack e Policy Decision.

4
Policy Check

Aplica políticas e exige evidências.

5
Evidence

Gera pacotes de evidências auditáveis.

O que o DevForge CLI faz

PRCP Scan

Detecta stack, risco e integrações externas.

Stack detectada:
- Next.js 15
- React 19
- FastAPI
- PostgreSQL
- Redis

Context Pack

Coleta contexto e metadados do repositório.

context/
├── project.md
├── arch.md
├── deps.md
└── index.json

Policy Gate

Aplica políticas organizacionais antes do merge.

Decisão:
REQUIRE_APPROVAL

Evidências
ausentes:
- human_review

Evidence Pack

Gera pacotes de evidências auditáveis.

evidence/
├── ISSUE-AUTH-001
│   ├── report.md
│   ├── audit.ndjson
│   └── diffs/

Local-first

Tudo rodando 100% local, sem cloud login.

Instale em segundos

pipx (recomendado)

$ pipx install git+https://github.com/mffdeo/devforge-ai-cli.git@v0.1.0

Pacote ainda não publicado no PyPI — instale direto do GitHub.

uv tool

$ uv tool install git+https://github.com/mffdeo/devforge-ai-cli.git@v0.1.0

Rápido e leve com o uv.

fonte (Git)

$ git clone https://github.com/mffdeo/devforge-ai-cli.git
